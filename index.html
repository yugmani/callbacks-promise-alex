  • Promises are objects that represent some value. You don't know that value when you create a Promise. You will get this value in the future when a promise is resolved or rejected.
  • When javaScript Promise is settled, resolved or rejected, it automatically triggers events you can listen to with specific handler methods.
  • The handlers will automatically execute any code you want at the right moment when Promise returns some value.

  • Creating a Promise

    1. While creating a Promise, we use Promise constructor using keyword 'new' followed by Promise().
    2. The constructor takes one parameter. This parameter is a function called executor. The executor is invoked automatically when you create a Promise.
    3. The executor function takes two parameters, both of which are callbacks. These parameters are resolve and reject.
    4. When Promise is resolved the resolve callback is invoked. This is when the job a Promise is supposed to do is successful. When it is not, when there is some error, Promise is rejected and the reject callback is invoked.
    5. // Promise syntax example
      const myPromise = new Promise(function(resolve, reject) {
        // ... some code
      })
    6. // Promise syntax example using an arrow function
      const myPromise = new Promise((resolve, reject) => {
        // ... some code
      })

  • Resolving and rejecting Promises with static methods

    When we want the Promise to return some data, ...

    1. we pass those data into the resolve. For example: while making a call to an API by our Promis, we pass into the resolve the data returned from the API.
    2. If some error occurs we can pass the data about the error to the reject callback. Or some error message.

  • Four States of JavaScript Promises

    1. First State: Pending - Thisis the initial state when you create a Promise and invoke it. This state says that the Promise is neither fulfilled(resolved) nor rejected.
    2. Second State: Fulfilled - This is the state that Promise is successfully resolved.
    3. Third State: Rejected - This is the state when there is some problem that is preventing the Promise from being successfully fulfilled(resolved).
    4. Fourth State: Settled - This is the last state of the Promise when the job of the Promise is finished and the Promise is either fulfilled or rejected.

  • Handling JavaScript Promises with Handler Functions

    1. Handling a Promise is working with the data returned by a Promise, data passed into resolve and reject callbacks.
    2. The then(), catch() and finally() are handler functions we can attach to a Promise.
    3. When you invoke a Promise and the promise is settled (resolved or rejected) one of these handlers will be automatically invoked. The Promise returns data and it is passsed into the handlers. The handlers are like messengers that transport the message from Promise further down the chain. For example: adding logic for updating an app with the data received from API into these handlers.

    4. The then() handler function

      1. This handler is usually used to handle fulfilled state of JavaScript Promises.
      2. When the Promise is settled(resolved or rejected) the data will be passed to the then() handler.
      3. These data are the data passed into the resolve() handler inside the Promise.
      4. To access this data, we need to pass a callback function to the then() handler. This callback function should accept one parameter. Any data returned by the Promise(resolved) will then be available through this parameter.
